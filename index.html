WORLD-LAYOFFS DATA CLEANING IN SQL

For SQL data cleaning project, I followed a structured approach to ensure the dataset was
accurate and well-organized. First, I identified and removed duplicate data to eliminate redundancy.
Then, I standardized formats to maintain consistency across all records. Next, I addressed null
values, fixing them wherever possible to improve data completeness. Finally, I removed unnecessary
columns and rows to streamline the dataset and keep only the most relevant information.

Throughout this process, several key SQL concepts applied are SELECT DISTINCT to filter out duplicate values
and CTEs (Common Table Expressions) to structure complex queries more clearly.
PARTITION BY is used to segment data efficiently, UPDATE TABLE to modify records, and
INNER JOIN to merge tables and extract meaningful insights.
